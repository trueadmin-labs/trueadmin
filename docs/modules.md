# TrueAdmin 模块规划

## 第一阶段：后台基础版

第一阶段先完成通用后台管理系统的基础能力设计，为后续企业增强模块铺好路。

## 认证与个人中心

- 登录。
- 退出登录。
- Token 刷新。
- 当前用户信息。
- 修改密码。
- 个人资料维护。

## 权限与组织

- 用户管理。
- 角色管理。
- 菜单管理。
- 按钮权限。
- 接口权限。
- 数据权限。
- 部门管理。
- 岗位管理。
- 组织树。

## 系统基础

- 字典管理。
- 参数配置预留。
- 文件上传。
- 操作日志。
- 登录日志。
- 基础仪表盘。

## 代码生成

第一阶段预留传统 CRUD 代码生成模块，目标是让开发者或 AI 根据元数据快速生成标准后台模块。

生成范围：

- 数据模型。
- 列表查询。
- 新增。
- 编辑。
- 删除。
- 详情。
- 导出预留。
- 菜单和权限点。
- Web 管理页面。

## 第二阶段：企业增强版

第二阶段在基础版稳定后扩展企业内部协作能力。

- 消息中心。
- 站内通知。
- 待办中心。
- 流程定义。
- 流程发起。
- 审批流转。
- 审批记录。
- 审批权限策略。

## 第三阶段：移动管理端

移动端定位为轻量管理端，优先服务管理人员处理移动事务。

- 登录。
- 消息列表。
- 待办列表。
- 审批详情。
- 审批操作。
- 个人中心。

## 暂不纳入第一版

- 多租户。
- 微服务拆分。
- 完整工作流引擎实现。
- 复杂低代码平台。
- BI 报表平台。

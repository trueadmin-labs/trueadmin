# AI 开发指南

## 目标

TrueAdmin 希望让 AI 能稳定参与企业后台开发。

这里的稳定不是指一次生成大量代码，而是指每次任务都有清晰输入、明确边界、可验证输出和可追溯变更。

## AI 开发基本流程

1. 阅读任务说明。
2. 阅读相关模块文档。
3. 阅读 API 规范和已有接口契约。
4. 明确本次修改范围。
5. 输出或更新模块设计。
6. 实现代码。
7. 补充测试或验证说明。
8. 更新文档和变更清单。

## 任务输入要求

每个任务建议包含：

- 背景：为什么要做。
- 目标：做到什么程度。
- 范围：允许改哪些目录或模块。
- 非目标：这次不做什么。
- 接口：涉及哪些 API。
- 权限：涉及哪些权限点。
- 数据：涉及哪些表或字段。
- 验收：如何确认完成。

## 模块开发要求

新增模块前，先补充模块说明。

模块说明至少包含：

- 模块名称。
- 模块职责。
- 主要实体。
- API 列表。
- 权限点。
- 数据表草案。
- Web 页面入口。
- 测试要点。

## 后端生成约定

AI 编写后端代码时，应遵循：

- Controller 只处理 HTTP 入参、鉴权入口和响应出口。
- Request 负责参数校验。
- Service 负责编排业务规则。
- Repository 负责数据查询和持久化。
- Model 不承载复杂业务流程。
- 权限点必须在接口说明中显式写出。
- 新增接口必须同步 OpenAPI 契约。

## Web 生成约定

AI 编写 Web 管理端代码时，应遵循：

- 页面放在业务模块目录内。
- 列表页统一包含查询、表格、分页和操作列。
- 新增与编辑优先复用表单结构。
- API 类型优先来自 OpenAPI 或共享 DTO。
- 按钮显示必须受权限点控制。
- 不在页面组件里散落硬编码接口路径。

## 移动端生成约定

AI 编写移动端代码时，应遵循：

- 移动端只承载轻量管理事务。
- 优先使用 Wot UI 组件。
- 页面交互适合单手操作和移动端信息密度。
- 复用后端认证和用户上下文。
- 审批、待办、消息等模块必须和 Web 端共享接口契约。

## 文档约定

不要把关键业务规则只留在对话里。

如果某个规则会影响后续开发，应写入这些位置之一：

- 模块文档。
- API 契约。
- 数据表说明。
- 权限点说明。
- AI 任务模板。

## 验证约定

AI 完成任务后，需要给出验证结果。

验证结果可以是：

- 单元测试。
- 接口测试。
- 类型检查。
- Lint。
- 页面手动验证。
- 暂时无法运行的原因。

## 不推荐的做法

- 未读文档直接写代码。
- 一次性跨多个模块做大范围修改。
- 后端字段、前端字段和文档字段命名不一致。
- 新增接口但不补契约。
- 新增权限按钮但不登记权限点。
- 修改业务规则但不更新文档。
